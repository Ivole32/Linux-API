## 2026-04-06

After I shipped, a few things happened. First, I want to clarify why I stopped working on the project. I was honestly a bit discouraged after receiving a reduced payout because the Fraud Squad flagged my project as some sort of AI-generated content. The long and frustrating discussion process drained my motivation, so I shifted my focus to other projects unrelated to Flavortown.

Now, onto the changes I’ve made:

* I polished the repository and added a security disclosure policy, along with issue templates.
* I upgraded the Python dependencies and spent some time reviewing the code for security issues.

Interestingly, I only found one issue. That could either mean I’m very good at writing secure code, or not so great at spotting vulnerabilities. Either way, the bug I did find could have had some impact and would have scored a low/medium CVSS score:

* Disabled routes were still being executed in the background, even though they returned a `503` status code.
* This happened because the access middleware performed validation in the wrong order.

This issue has now been fixed.

Additionally, I discovered that the rate limiter was disabled by default, which I’ve also corrected.
